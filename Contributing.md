# Contributing to Macad|3D

First of all, thank you for your interest in contributing to **Macad|3D**! Every contribution - whether it's code, documentation, testing, or community support - helps make the project better.

## Ways to Contribute

Not every contribution requires writing code. Here are some ways you can help:

- Find and report bugs with clear reproduction steps.
- Improve the documentation by contributing to the [User Guide](https://github.com/Macad3D/UserGuide)
- Promote Macad3D by sharing it with others, writing tutorials, creating videos, or showcasing projects created with it.
- Create tutorials, answer questions and help other users in discussions and issue threads.
- Submit bug fixes or new features through pull requests.

Every contribution is appreciated, regardless of its size.

## Before You Start

- Check existing issues and pull requests to avoid duplicate work.
- If you're planning a significant change, please open an issue first to discuss the design and implementation.
- Review the [User Guide](https://github.com/Macad3D/UserGuide) to become familiar with the project.

## Development Guidelines

- Follow the project's coding style and conventions.
- Add or update tests for any new functionality or bug fixes.
- Update the documentation in the [User Guide](https://github.com/Macad3D/UserGuide) whenever behavior or user-facing functionality changes.
- Ensure all tests pass before submitting a pull request.

## Pull Requests

When submitting a pull request:

- Provide a clear description of the problem being solved.
- Explain your approach and any design decisions.
- Reference related issues where applicable.
- Keep commits focused and use meaningful commit messages.

Please keep pull requests reasonably small and focused. Smaller changes are easier to review and are more likely to be merged quickly.

## AI-Assisted Development

AI coding assistants are welcome as productivity tools. However, contributors are expected to fully understand, review, and take responsibility for every change they submit.

Please strictly adhere to the following rules:
- Do not use AI to write your Pull Request descriptions.
- Never submit code that you do not fully understand. 
- No copy-paste garbage: AI models frequently generate overly complex code. Manually refine, clean, and optimize all AI suggestions before committing.
- Your code must seamlessly fit our existing architecture and formatting guidelines.
- Ensure your AI tool did not copy code from repositories with incompatible open-source licenses (e.g., GPL). All submissions must legally comply with our MIT License.

Maintainers have limited time, so every minute spent reviewing and rewriting AI code means less time for designing new features.
Therefore, pull requests will be closed immediately if there is reason to believe that the code or description was generated by AI without thorough human review, testing, and refinement.
